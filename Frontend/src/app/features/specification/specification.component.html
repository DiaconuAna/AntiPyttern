Design Defect Specifications

Blob (aka God Class, Brain Class)

The Blob antipattern manifests when one class dominates control over an application's behavior. It typically consists of a bloated controller class paired with passive data classes.

In Python, it's usually a large class with many methods and attributes, leading to low cohesion and violating the Single Responsibility Principle.

Underlying Code Smells

Data Class Component: Data Class

Controller Component:

  • Large Class
  • Long Method
  • Switch Statement
  • Temporary Field
  • Feature Envy
  • Inappropriate Intimacy
  • Divergent Change
  • Shotgun Surgery
  • Data Clumps
  • Comments

In Python, it shows up in long, multi-purpose functions, global variables, and a lack of classes or modular design, often resulting in large monolithic scripts.

Underlying Code Smells

  • Long Method with no parameters
  • Feature Envy
  • Message Chains
  • Switch Statements
  • Dead Code and Speculative Generality

Swiss Army Knife

High-level design defect that occurs at class level. It refers to a class that attempts to provide a variety of unrelated functionalities, thereby assuming an excessive number of responsibilities.

In Python it manifests through inheritance from multiple abstract base classes or mixins, a large number of unrelated methods, intended for speculation or utility, and is characterised by low cohesion and violation of the Single Responsibility Principle.

Underlying Code Smells

  • Large Class with Too Many Methods
  • Feature Envy
  • Divergent Change
  • Speculative Generality
